# Event Safety Checklist

## 트리거

- safety-checker Agent가 `01-checklist-roles.md`, `02-schedule.md`를 최초 검토할 때
- D-1 자동 재점검(cron)으로 safety-checker가 재실행될 때

## 절차

1. `references/standing-rules.md`의 "안전 필수 확인 항목"을 기준 목록으로 불러온다.
2. 아래 항목을 `01-checklist-roles.md`, `02-schedule.md`와 대조한다.

   | 항목 | 확인할 것 |
   | --- | --- |
   | 응급 연락처 | 담당자·인근 병원 연락처가 준비물 또는 일정에 명시되어 있는가 |
   | 알레르기·식이 | 식사/케이터링 항목에 참석자 알레르기 확인 절차가 있는가 |
   | 기상 대안 | 실외 요소가 있다면 우천/혹서 대안이 일정에 있는가 |
   | 이동 수단 안전 | 버스/자차 이동이 있다면 음주운전 금지·안전벨트 고지가 있는가 |
   | 화재·대피 | 실내 대관 장소라면 대피 동선 확인 항목이 있는가 |
   | 안전 인력 비율 | 인원 대비 담당 인력(응급대응 포함)이 배정되어 있는가 |
   | 보험 | 행사 보험 가입 여부가 확인 필요 항목으로라도 표시되어 있는가 |

3. 항목마다 통과/보완 필요를 판정하고, 판정 근거(어떤 문장·항목과 대조했는지)를 남긴다.
4. 보완 필요 항목이 있으면 담당 Agent에게 `SendMessage`로 구체적으로 요청한다(최대 2회 루프).

## 산출물 형식

```md
# 안전 점검 결과 ({최초 | D-1 재점검})

| 항목 | 판정 | 근거 | 확인 필요 |
| --- | --- | --- | --- |
| 응급 연락처 | 통과/보완 필요 |  |  |
| 알레르기·식이 |  |  |  |
| 기상 대안 |  |  |  |
| 이동 수단 안전 |  |  |  |
| 화재·대피 |  |  |  |
| 안전 인력 비율 |  |  |  |
| 보험 |  |  |  |

## 종합 판정
- 전체 상태: 승인 가능 | 보완 필요 | 사람 확인 필요
- 미해결 항목:
```

## 품질 체크

- 모든 항목에 근거가 있는가 (판정만 있고 근거가 빈 칸이면 재작성)
- 보완 필요 항목이 하나라도 있으면 종합 판정을 "승인 가능"으로 쓰지 않았는가
- D-1 재점검은 최초 점검과 같은 표를 다시 채우고, 최초 점검 이후 바뀐 부분(예: 인원 변경, 장소 변경)을 별도로 표시했는가
